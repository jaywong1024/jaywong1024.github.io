    一、缓存穿透

    1.1 概述

    缓存穿透 指的是有大量请求的 key 是不合理的（根本不存在于 cache 和 db 的数据），这会导致 db 书山压力大，很有可能就直接宕机了，场景如下

    为了防止黑客对我们发起非法请求（伪造大量不存在的 key），并且通过安全测试（与国际公司合作时，他们会聘请专业的测试团队，其中就有可能造成缓存穿透的测试），可以使用下面的两个办法处理缓存穿透

    1.2 缓存无效的 key（不推荐）

    如果 cache 和 db 都找不到某个 key 的数据，就将无效的 key 缓存到 cache 中。这种方案适用于 key 变化不频繁的情况，如果频繁构建不同 key 进行大量请求，这种方案显然是不适用的，会造成 cache 中一堆脏数据的情况。所以如果要使用这种方案的话，建议将 key 的过期时间设置的较短

    1.3 布隆过滤器（推荐）

    1.3.1 概述

    布隆过滤器（Bloom Filter）是 1970 年由布隆提出的。它实际上是一个很长的二进制向量（BITS）和一系列随机映射函数。布隆过滤器可以用于检索一个元素是否在一个集合中

    1.3.2 工作原理

    主要用来对存储的数据做 hash 计算，得到对应的散列值，这个散列值就是这个值对应的在数组的位置下标，每一个 hash函数做一次散列计算，得到这个要存储的值的一系列下标位置，然后把对应位置的数据里面的数据修改为 1，表示该数据存储进入布隆过滤器。查询数据的时候只需要对数据做对应的散列计算，然后比对下标中的值，如果值全部为 1 则表示这个数据是存在布隆过滤器的，只要有一个为 0，就表示布隆过滤器中没有该数据。这即是布隆过滤器的工作原理

    综上，我们可以得出：布隆过滤器说某个元素存在，小概率会误判。布隆过滤器说某个元素不在，那么这个元素一定不在

    1.3.5 Redis 中的布隆过滤器

    Redis v4.0 之后有了 Module（模块/插件） 功能，Redis Modules 让 Redis 可以使用外部模块扩展其功能 。布隆过滤器就是其中的 Module。官网推荐了一个 RedisBloom 作为 Redis 布隆过滤器的 Module

    命令

    • BF.ADD：将一个元素添加到布隆过滤器中，如果过滤器不存在，则创建该过滤器
    • BF.MADD : 将一个或多个元素添加到布隆过滤器中
    • BF.EXISTS : 确定元素是否在布隆过滤器中
    • BF.MEXISTS ： 确定一个或者多个元素是否在布隆过滤器中

    BF.RESERVE {key} {error_rate} {capacity}

    • key：布隆过滤器的名称
    • error_rate：最大误判率率，例如千分之一为 0.001（该数字越接近零，则每个项目的内存消耗越大，并且每个操作的 CPU 使用率越高）
    • capacity：过滤器的容量。当实际存储的元素个数超过这个值之后，性能将开始下降。实际的降级将取决于超出限制的程度。随着过滤器元素数量呈指数增长，性能将线性下降

    二、缓存击穿

    2.1 概述

    缓存击穿和缓存穿透的同样大量请求的场景，不同点在于，缓存击穿请求的 key 对应的是 热点数据 ，该数据 存在于db 中，但不存在于 cache 中（通常是因为缓存中的那份数据已经过期）。这可能导致大量请求直接打到 db 上，对数据库造成巨大压力，然后导致 db 宕机

    例如秒杀商品数据的缓存过期了，大量请求商品信息的请求直接到了 db，对 db 造成大量压力

    2.2 解决方案

    • 设置热点数据永不过期，或者设置较长的过期时间
    • 针对热点数据提前进行缓存，将其存入 cache 中并设置合理过期时间（例如热点商品在活动后才过期）
    • 请求数据库写数据到缓存前，先获取互斥锁，保证只有少部分请求会落到数据库中

    三、缓存雪崩

    3.1 概述

    缓存雪崩指的是 cache 中大量数据在同一时间大面积的失效，导致大量的请求到直接打到了 db 上，对数据库造成巨大压力，可能导致 db 宕机。缓存雪崩和缓存击穿不同的是，缓存雪崩是 cache 中大量数据或者所有数据同时失效导致的，而缓存击穿是某个热点数据不存在导致的

    例如通过系统初始化的数据，它们的过期时间近似，在过期后突然有大量请求尝试获取这批数据

    3.2 解决方案

    • 缓存数据的过期时间设置随机，防止同一时间大量数据过期现象发生
    • 如果缓存数据库是分布式部署，将热点数据均匀分布在不同搞得缓存数据库中
    • 设置热点数据永远不过期
