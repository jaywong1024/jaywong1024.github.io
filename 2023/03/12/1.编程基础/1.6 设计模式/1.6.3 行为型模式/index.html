    1.6.3【编程基础 - 设计模式】行为型模式

    行为型模式：关注的是各个类之间的相互作用，将职责划分清楚，使得我们的代码更加地清晰

    一、策略模式

    1.1 意图

    • 将算法封装在独立的类中，使得它们可以相互独立地变化，而不会影响到客户端的使用。通过这种方式，我们可以实现高度的灵活性和可扩展性，因为客户端可以根据需求动态地选择不同的算法

    1.2 主要解决

    • 在不同的场景下，需要采用不同的算法或策略，但是不希望客户端依赖于具体的算法实现
    • 需要对算法进行扩展或修改，但是不希望影响到客户端的代码
    • 需要动态地切换算法或策略

    1.3 何时使用

    • 在不同的场景下需要采用不同的算法或策略，并且这些算法或策略的实现可能会发生变化
    • 需要动态地切换算法或策略，并且不希望对客户端的代码造成影响

    二、观察者模式

    2.1 意图

    • 在对象间定义一种一对多的依赖关系，使得当一个对象状态发生改变时，所有依赖于它的对象都能够得到通知并自动更新

    2.2 主要解决

    • 当一个对象状态发生改变时，需要通知其他对象，并且需要尽可能避免对象间的紧密耦合关系。通过使用观察者模式，可以实现松耦合的对象间通信，使得对象间更易于扩展和维护

    2.3 何时使用

    • 一个对象的改变需要同时改变其他对象的状态，而且它们的数量不确定；
    • 一个对象的改变需要改变另一个对象，但是不需要知道是哪个对象；
    • 对象间的耦合度过高，需要解耦

    三、责任链模式

    3.1 意图

    • 将请求从一个对象传递到另一个对象，直到请求被处理为止

    3.2 主要解决

    • 当一个对象状态发生改变时，需要通知其他对象，并且需要尽可能避免对象间的紧密耦合关系。通过使用观察者模式，可以实现松耦合的对象间通信，使得对象间更易于扩展和维护

    3.3 何时使用

    • 当系统中存在多个对象可以处理请求，并且请求的处理可能需要按照一定的顺序进行时，可以考虑使用责任链模式

    四、模板方法模式

    4.1 意图

    • 定义一个算法的骨架，将一些步骤的具体实现延迟到子类中

    4.2 主要解决

    • 解决了在一个算法框架中，有一些步骤是固定的，而另一些步骤是可变的，这时候我们可以将固定的步骤封装在一个模板方法中，而将可变的步骤交给子类去实现

    4.3 何时使用

    • 算法骨架不变，但是其中的一些步骤可以有不同的实现方式
    • 子类需要实现一些公共行为，但又不希望子类直接修改算法的结构
    • 需要控制子类的扩展，防止子类改变算法的顺序
    • 需要封装固定的算法流程，隐藏算法的实现细节

    4.5 开源例子

    • HttpServlet：HttpServlet 是 Java Web 中的核心类之一，它提供了处理HTTP请求的模板方法 doGet、doPost 等。在开发 Web 应用时，可以继承 HttpServlet 并重写其中的模板方法，实现自己的业务逻辑
    • JDBC Template：JdbcTemplate 是 Spring 框架中用于简化 JDBC 操作的核心类之一，它提供了一系列的模板方法，如 query、update 等，让开发者可以在不关心 JDBC 具体实现的情况下进行数据库操作
    • JUnit TestCase：JUnit 是 Java 中广泛使用的单元测试框架，其中 TestCase 类定义了许多模板方法，如 setUp、tearDown、runTest 等，开发者可以在子类中重写这些方法，实现测试用例的编写
    • AQS 是 Java 中并发包中用于构建锁和同步器的基础类，它提供了一个可扩展的同步器框架，其中的锁算法通过模板方法模式实现。AQS 定义了如下几个模板方法：
      • tryAcquire：尝试获取锁，需要子类实现。
      • tryRelease：尝试释放锁，需要子类实现。
      • tryAcquireShared：尝试获取共享锁，需要子类实现。
      • tryReleaseShared：尝试释放共享锁，需要子类实现。
      • isHeldExclusively：查询当前线程是否独占锁，需要子类实现

    六、总结

    • 策略模式：定义一系列算法，将它们封装起来，并使它们可以相互替换。让算法的变化独立于使用算法的客户端。
    • 观察者模式：定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象。当主题对象发生变化时，它的所有依赖者（观察者）都会收到通知并自动更新。
    • 责任链模式：将请求的发送者和接收者解耦，让多个接收对象都有机会处理这个请求。并将这些接收对象串成一条链，沿着这条链传递请求，直到有对象处理它为止。
    • 模板方法模式：定义一个操作中的算法的骨架，而将一些步骤延迟到子类中。使得子类可以不改变一个算法的结构即可重定义该算法的某些特定步骤
