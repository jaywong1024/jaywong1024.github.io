    1.1.2.3【编程基础 - Java 语言 - JVM 虚拟机】字节码执行机制

    Java 语言是一门编译型语言，编写的 Java 程序需要经过编译器编译成二进制字节码文件，然后由 Java 虚拟机来执行这些字节码。Java 字节码执行机制是 Java 程序运行的基础

    一、字节码文件

    Java 字节码文件是 Java 程序编译后生成的二进制文件，它包含了 Java 程序中的所有信息，包括变量、方法、类等。Java字节码具有跨平台的特性，可以在不同的操作系统和硬件平台上运行

    1.1 魔数（4个字节）

    每个 Class 的文件开始的 4 个字节被称为魔数（Magic Number），字节码值为 CAFEBABE，JVM 通过校验魔数来判断文件是否为可运行的字节码文件，这种方式比扩展名更安全

    1.2 版本号（4个字节）

    紧接着魔数的 4 个字节存储的是字节码文件的版本号，前两个字节为次版本号（Minor Version，现在基本都不用了），后两个字节为主版本号，例如上面的 0000 0034 转换为 10 进制为 52，52 代表此字节码文件由 JDK 8 编译而成（以此类推，51 为 JDK 7，50 为 JDK 6，53 为 JDK 9）

    1.3 常量池（N个字节）

    1.3.1 常量池计数（2个字节）

    在版本号之后的 2 个字节存储的是常量池计数（constant_pool_count），它用于描述常量池中常量的数量（数量为 constant_pool_count - 1，因为计数是从 1 开始算的）

    1.3.3 标识字符表

    标识字符 含义
    B 基本类型 byte
    C 基本类型 char
    D 基本类型 double
    F 基本类型 float
    I 基本类型 int
    J 基本类型 long
    S 基本类型 short
    Z 基本类型 boolean

    字段或方法的部分符号引用（CONSTANT_NameAndType_info） 类型会使用到标识字符，下面给出一些例子

    • ()V 表示返回值为 void 无参方法，void method()
    • (Ljava/lang/Object;Ljava/lang/Object;)Ljava/lang/Object 表示返回值为 Object 并且入参为两个 Object 的方法（注意对象类型作为参数，必须在后面添加 ;，Object method(Object obj1, Object obj2)
    • ([[I;)V 表示入参为二维数组（有几个维度就加几个 [），返回值为 void 的方法，void method(int[][] arr)

    1.3.4 查看常量池

    常量池计数值后就是常量值了，例如 Test 类的常量为 0033 -> 51 - 1 = 50 个

    第 1 个常量 #1 的 tag 为 0a -> 10，类型为 类中方法的符号引用（CONSTANT_Methodref_info），那么可以得知长度为 u1 + u2 + u2，字节码为 0a 000a 0021，分析可得 0a -> 10, 000a -> #10, 0021 -> #33，#1 指向 #10 和 #33

    同理，依次找到 #10，类型为 类或接口的符号引用（CONSTANT_Class_info ），它指 #43

    • 得到 #43 为 java/lang/Object

    找到 #33，类型为 字段或方法的部分符号引用（CONSTANT_NameAndType_info），它指向 #14 和 #15

    • 得到 #14 为 <init>， #15 为 ()V

    拼凑起来可以得到 #1 为 java/lang/Object."<init>":()V，意思就是 Object 类的构造方法（构造方法名称），返回值为 Void（所有的类都是继承 Object 类的）

    1.4 访问标志（2个字节）

    找到 Test.class 字节码文件中最后一个常量，在 000001f8 行的 01 0004 28492956 中，表示一个 UTF-8 字符串，长度为 4，16 进制 28492956 转译字符串为 (I)V。

    紧接着在它后面的就是访问标志 0021（由多个标志值进行相加运算），例如 0x0021 = 0x0001 + 0x0020，它代表 Test 类的访问标志为 public

    标志名称 标志值 含义
    ACC_PUBLIC 0x0001 是否被 public 修饰
    ACC_FINAL 0x0010 是否被 final 修饰
    ACC_SUPER 0x0020 是否允许使用 invokespecial 字节码指令的新语意，invokespecial 指令的语意在 JDK 1.0.2 发生过改变，为了区别这条指令使用哪种语意，JDK 1.0.2 之后编译出来的类的这个标志都必须为真
    ACC_INTERFACE 0x0200 标识这是一个接口
    ACC_ABSTRACT 0x0400 是否为 abstract 类型，对于接口或者抽象类来说，此标志值为真，其他类值为假
    ACC_SYNTHETIC 0x1000 标识这个类并非由用户代码产生的
    ACC_ANNOTION 0x2000 标识这是一个注解
    ACC_ENUM 0x4000 标识这是一个枚举
    ACC_MODULE 0x8000 标识这是一个模块

    1.5 类信息（4个字节）

    紧接着访问标志的就是类信息（本类和父类），Test.class 中为 0008 000a 分别指向常量 #8 -> #42 和 #10 -> #43，#42 本类为 top/hanjie/jvm/classinfo/Test，#43 父类为 java/lang/Object

    1.6 接口（N个字节）

    类信息后 2 个字节储存着接口计数（interfaces_count），它用于描述类实现的接口数量，Test.class 中为 1 个，它的值就在后面 000b -> #11 -> #44 为 top/hanjie/jvm/classinfo/TestApi

    1.7 字段（N个字节）

    1.7.1 字段计数（2个字节）

    接口信息后 2 个字节储存着字段计数（fields_count），它用于描述字段的数量，Test.class 中为 1 个

    1.7.2 字段结构（8个字节）

    字段结构使用 8 个固定的字节来描述，在 Test.class 中，001a 000c 000d 0000 就是它的唯一字段

    • 字段访问标志 001a = 0x0002 + 0x0008 + 0x0010 = private static final
    • 字段名称 000c -> #12 -> test
    • 描述符 000d -> #13 -> Ltop/hanjie/jvm/classinfo/Test;
    类型 名称 数量 备注
    u2 access_flags 1 字段访问标志
    u2 name_index 1 字段名称
    u2 descriptor_index 1 字段和方法的描述符
    u2 attributes_count 1 字段没有附加属性（所以为 0）
    attribute_info attributes attributes_count 无

    1.7.3 字段访问标志

    标志名称 标志值 含义
    ACC_PUBLIC 0x0001 字段是否被 public 修饰
    ACC_PRIVATE 0x0002 字段是否被 private 修饰
    ACC_PROTECTED 0x0004 字段是否被 protected 修饰
    ACC_STATIC 0x0008 字段是否被 static 修饰
    ACC_FINAL 0x0010 字段是否被 final 修饰
    ACC_VOLATILE 0x0040 字段是否被 volatile 修饰
    ACC_TRANSIENT 0x0080 字段是否被 transient 修饰

    1.8 方法（N个字节）

    1.8.1 方法计数（2个字节）

    字段信息后 2 个字节储存着方法计数（methods_count），它用于描述字段的数量，Test.class 中为 6 个

    1.8.2 方法结构

    方法结构使用 8 个固定的字节来和动态字节的属性集合描述，在 Test.class 中分别为

    • ：构造方法
    • main：主方法
    • a：普通方法
    • b：普通方法
    • c：普通方法
    • ：优先于构造方法，在类加载后调用
    类型 名称 数量 备注
    u2 access_flags 1 方法访问标志
    u2 name_index 1 方法名称
    u2 descriptor_index 1 字段和方法的描述符
    u2 attributes_count 1 方法中包含属性数量
    attribute_info attributes attributes_count 属性有代码、参数等

    1.8.3 方法访问标志

    ACC_PUBLIC 0x0001 字段是否被 public 修饰
    ACC_PRIVATE 0x0002 方法是否被 private 修饰
    ACC_PROTECTED 0x0004 方法是否被 protected 修饰
    ACC_STATIC 0x0008 方法是否被 static 修饰
    ACC_FINAL 0x0010 方法是否被 final 修饰
    ACC_ABSTRACT 0x0400 方法是否被 abstract 修饰

    1.9 属性（N个字节）

    1.9.1 属性计数（2个字节）

    方法信息后 2 个字节储存着属性计数（attributes_count），它用于描述字段的数量，Test.class 中为 1 个

    1.9.2 属性结构

    属性名称 使用位置 含 义
    Code 方法表 Java 代码编译成的字节码指令
    Constant Value 字段表 final 关键字定义的常量值
    Deprecated 类、方法表、字段表 被声明为 deprecated 的方法和字段
    Exceptions 方法表 方法抛出的异常
    EnclosingMethod 类文件 仅当一个类为局部类或者匿名类时才能拥有这个属性，这个属性用于标识这个类所在的外围方法
    InnerClasses 类文件 内部类列表
    LineNumberTable Code 属性 Java 源码的行号与字节码指令的对应关系
    LocalVariableTable Code属性 方法的局部变量描述
    StackMapTable Code属性 JDK 6 中新增的属性，供新的类型检查验证器（Type Checker）检查和处理目标方法的局部变量和操作数栈所需要的类型是否匹配
    Signature 类、方法表、字段表 JDK 5 中新增的属性，这个属性用于支持泛型情况下的方法签名，在Java 语言中，任何类、接口、初始化方法或成员的泛型签名如果包含了类型 变量（Type Variables）或参数化类型（Parameterized Types），则 Signature 属性会为它记录泛型签名信息。 由于 Java 的泛型采用擦除法实现，在为了避免类型信息被擦除后导致签名混乱，需要这个属性记录泛型中的相关信息
    SourceFile 类文件 记录源文件名称
    SourceDebugExtension 类文件 JDK 1．6中新增的属性，SourceDebugExtension属 性用于存储额外的调试信息。譬如在进行JSP文件调 试时，无法通过 Java 堆栈来定位到JSP文件的行号， JSR-45规范为这些非Java 语言编写，却需要编译成 字节码并运行在Java虚拟机中的程序提供了一个进 行调试的标准机制，使用 SourceDebugExtension 属性 就可以用于存储这个标准所新加入的调试信息
    Synthetic 类、方法表、字段表 标识方法或字段为编译器自动生成的
    LocalVariableTypeTable 类 JDK 5 中新增的属性，它使用特征签名代替描述符，是为了引入泛型语法之后能描述泛型参数化类型而添加
    RuntimeVisibleAnnotations 类、方法表、字段表 JDK 5 中新增的属性，为动态注解提供支持。 RuntimeVisibleAnnotations 属性用于指明哪些注解是运行时（实际上运行时就是进行反射调用）可见的
    RuntimelnvisibleAnnotations 类、方法表、字段表 JDK 5 中新增的属性，与 RuntimeVisibleAnnotations 属性作用刚好相反，用于指明哪些注解是运行时不可 见的
    RuntimeVisibleParameter Annotations 方法表 JDK 5 中新增的属性，作用与RuntimeVisibleAnnotations 属性类似，只不过作用对象为方法参数
    RuntimelnvisibleParameter Annotations 方法表 JDK 5 中新增的属性，作用与 RuntimelnvisibleAnnotations 属性类似，只不过作用对象为方法参数
    AnnotationDefault 方法表 JDK 5 中新增的属性，用于记录注解类元素的默 认值
    BootstrapMethods 类文件 JDK 7 中新增的属性，用于保存 invokedynamic 指令引用的引导方法限定符

    属性结构如上表，在 Test.class 中，为 0000 000c 0001 001f 0000 0002 0020，使用 jclasslib 可以看到属性名称为 SourceFile，代表它是用于记录源文件名称的属性，特有信息中记录了源文件名称为 Test.java

    二、字节码执行过程

    Java 字节码执行过程可以分为三个阶段：

    1. 加载阶段：Java 虚拟机将字节码文件加载到内存中，并对字节码进行验证、准备和解析
    2. 运行阶段：Java 虚拟机执行字节码文件，并将字节码解释成机器码运行
    3. 卸载阶段：Java 虚拟机在程序执行完成后，释放内存中的字节码文件

    三、字节码执行优化

    Java 字节码执行过程中，为了提高程序的执行效率，Java虚拟机采用了一系列的优化措施，包括：

    1. 即时编译（Just-In-Time Compilation，JIT）：Java 虚拟机将频繁执行的字节码编译成本地机器码，提高程序的执行速度
    2. 垃圾回收（Garbage Collection，GC）：Java 虚拟机自动管理内存，释放不再使用的内存空间，避免内存溢出
    3. 类加载优化：Java 虚拟机采用类加载器来加载字节码文件，可以对类加载器进行优化，提高程序的执行效率

    四、总结

    Java 字节码执行机制是 Java 程序运行的基础，Java 虚拟机负责将 Java 字节码解释成机器码并执行。为了提高程序的执行效率，Java 虚拟机采用了一系列的优化措施，包括即时编译、垃圾回收和类加载优化等。掌握Java字节码执行机制对于Java程序员来说是非常重要的
