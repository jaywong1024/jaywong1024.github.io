    1.1.2.1【编程基础 - Java 语言 - JVM 虚拟机】内存模型

    一、内存区域划分

    既然要管理内存，那么肯定不会是杂乱无章的，JVM 对内的管理采用的是分区治理，不同的内存区域有着各自的职责所在，在虚拟机运行时，内存区域如下划分（仅概念模型，各种虚拟机有各自不同的实现方式）：

    1.1 程序计数器（Program Counter Register）

    程序计数器是一块较小的内存空间，它可以看作是当前线程所执行的字节码的行号指示器。在虚拟机概念模型里，字节码解释器工作时就是通过改变程序计数器的值来选取下一条需要执行的字节码指令：分支、跳转、循环、异常处理、线程恢复等基础操作都会依赖程序计数器来完成。

    每个线程都有独立的程序计数器，用来在线程切换后能恢复到正确的执行位置。各条线程之间的程序计数器互不影响，是一个 线程私有 的内存区域。此内存区域是唯一一个在 JVM 规范中没有规定任何 OutOfMemoryError 情况的区域

    1.2 虚拟机栈（VM Stack）

    1.2.1 定义

    虚拟机栈是线程私有的内存区域。每个方法执行的时候都会创建一个 栈帧（Stack Frame）用于存储 局部变量表、操作数栈、动态链接、方法出口 等信息。每个方法从调用至完成的过程，都对应着一个栈帧从入栈到出栈的过程。每当一个方法执行完成时，就会弹出该栈帧的元素作为这个方法的返回值，并且将栈帧在 虚拟机栈 中移除（出栈）。虚拟机栈的栈顶，就是当前正在执行的活动栈，也就是当前正在执行的方法

    • 局部变量表：存放了编译器可知的基本数据类型（int、short、long、float、double、byte、char、boolean）、对象引用（reference 引用类型）和 returnAddress 类型（指向下一跳指令内存地址的指针）
    • 操作数栈：主要用于保存计算过程的中间结果，同时作为计算过程中变量临时的存储空间
      • 每一个操作数栈在创建（随着栈帧创建）时都是空的，但会有一个明确的栈深用于存储数值，其所需的最大深度在编译期间就定义好了，保存在方法的 Code 属性中，为 MAXSTACK 的值
      • 操作数栈使用的是标准的入栈出栈操作来完成数据的暂存和访问
      • 32bit 类型的数据占用 1 个栈单位深度，64bit 类型的数据占用 2 个栈单位深度
    • 动态链接：在 Java 源文件被编译成字节码文件时，所有变量和方法引用都作为符号引用（Symbolic Reference）保存在 .class 文件的常量池中。在字节码中描述一个方法调用其他方法时，就是通过常量池中的符号引用来表示的。如果一个被调用方法在编译期间可知，且允许时保持不变，这种情况下将调用方法的符号引用转换为直接引用的过程称之为 静态链接。反之，如果被调用方法无法在编译期间确认下来，只能由程序运行期间将调用方法的符号引用转换为直接引用，由于这种引用转换过程中具备动态性，因此被成为 动态链接。例如接口回调，多态动态绑定等
    • 方法出口：方法出口指的是执行完成方法后，返回到方法被调用的位置

    在 JVM 规范中，对这个区域规定了两种异常情况

    • 如果线程请求的栈深度大于虚拟机允许的深度，将抛出 StackOverflowError 异常
    • 如果虚拟机栈可以动态扩展，在扩展时无法申请到足够的内存，将抛出 OutOfMemoryError 异常

    1.3 本地方法栈（Native Medhod Stack）

    本地方法栈和虚拟机栈的作用是很相似的，它们之间的区别不过是虚拟机栈为虚拟机执行 Java 方法（字节码）服务，而本地方法栈则为虚拟机使用到的 Native 方法服务。Sun HotSpot 直接就把本地方法栈和虚拟机栈合二为一

    本地方法栈也会抛出 将抛出 StackOverflowError 异常和 OutOfMemoryError 异常

    1.4 堆（Heap）

    堆是所有线程共享的内存区域，它存储着几乎所有的实例对象，堆中的对象由垃圾收集器自动回收。通常情况下，堆占用的空间是整个内存模型中最大的，但如果无节制的创建大量对象，也容易消耗完所有的空间，所以是 OOM 故障最主要的发源地（GC 后内存还是塞不下对象了）

    堆的内存空间既可以固定大小，也可以运行时动态的调整，通过参数 -Xms 设置初始值，-Xmx 设置最大值

    1.5 方法区（Method Area）

    方法区是所有线程共享的内存区域，用来存储已经被虚拟机加载的 类信息、常量、静态变量、JIT（即使编译技术）编译后的代码等数据。运行时常量池也是方法区的一部分，用于存放编译期间生成的各种字面常量和符号引用

    1.6 总结

    以上内存区域划分仅概念模型，各种虚拟机有各自不同的实现方式，下面总结一下各个内存区域的用途

    • 程序计数器（线程隔离）：保存当前线程程序执行的位置
    • 虚拟机栈（线程隔离）：通过栈帧来维持方法的调用顺序，保存方法调用必要的信息
    • 本地方法栈（线程隔离）：同上，但只作用于本地方法
    • 堆（线程共享）：存储着几乎所有的实例对象
    • 方法区（线程共享）：保存类信息、常量、静态变量、JIT 等

    三、直接内存

    3.1 概述

    除了堆内存可以存放数据以外，还可以申请堆外内存（直接内存）来存放数据，这部分内存区域不受 JVM 管控，需要我们自行去申请和释放内存（实际上就是 JVM 通过 C/C++ 调用 malloc 函数申请内存，那当然得自己去释放内存）。虽然申请直接内存不会受到 JVM 堆大小的限制，但依旧会受到本机最大内存的限制，所以还是有可能出现 OOM 异常的
