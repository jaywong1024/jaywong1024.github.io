    二、サービスの学習

    中国の IT インフラストラクチャサービス（Infrastructure Services）の Aliyun と Tencent Cloud を使用したことがあるので、経験とドキュメント（Document）に基づいて直接使用してみるつもりです。

    1.1 インスタンス（Instance）

    1.1.1 概要

    EC2 は AWS の基本サービスであり、シェル（Shell）を介してリモート（Remote）から直接アクセス（Access）でき、データベース（Database）、キャッシュ（Cache）、メッセージ キュー（Message Queue）などをインストールできます。ただし、通常は AWS カスタム（Custom） データベース、キャッシュ、メッセージ キューが使用されます。 EC2 は、カスタム開発された Web プログラム（Program）を実行するために最も一般的に使用されます。AWS では、クバネティス（Kubernetes）と同様に、トラフィック（Traffic：ネットワーク上を流れる情報量）やポリシー（Policy）に応じて EC2 インスタンスを調整できるエラスティック スケーリング サービス（Elastic Scaling Services）を提供しており、非常に便利です。

    Amazon EC2 ドキュメント（Document）：https://docs.aws.amazon.com/ja_jp/whitepapers/latest/aws-overview/compute-services.html#amazon-ec2

    1.1.2 作成

    「名前」に「jay-ec2-01」を入力して、「アプリケーション（Application）および OS イメージ（Image）」が「Amazon Linux」を選択しました。

    「新しいキーペア（Key pair）の作成」をクリックしました。

    「キーペア名」に「jay-kp-rsa-01」を入力しました。

    「キーペアを作成」をクリックすると、ブラウザ（Browser）はプライベートキー（Private Key）を pem 形式で自動的にダウンロードしました。

    新しいキーペアを選択しました。

    デフォルトの 8GiB を 25GiB に変更しました。

    他のすべてのデフォルト（Default）、「インスタンスを起動」をクリックして、作成が完了します。

    1.1.3 接続

    インスタンスの初期化が完了したら、「パブリック IPv4 アドレス」と「ダウンロードしたプライベートキー」を使用してアクセスします。

    接続には SSH クライアントを使用して、個人的には Tabby を使用するのが好きです。

    接続完成。

    テストのために Nginx をインストールします。

    テストのために新しい HTML と Nginx の設定を作成します。

    2.2 データベース

    2.2.1 概要

    まず、第一にリレーショナル（Relational） データベース、ほとんどのサービスの基礎となるから。Amazon は MySQL および PostgreSQL 互換のリレーショナルデータベースの Amazon Aurora（有料）を提供しています。

    個人的な経験によれば、安定性が最も重要であるため。だから、その中で、「最大 15 個の低レイテンシーのリードレプリカ、 point-in-time リカバリ、Amazon S3 への継続的なバックアップ」がとても重要の機能だと思います。Samsung、Nintendo、A+E Networks などの企業が Amazon Aurora を使用しており、これは安定性を反映しています。

    2.2.2 作成

    MySQL は無料のデバッグを提供するため、個人の開発者の場合は MySQL を選択します。

    「DB クラスター識別子」に「jay-db-01」を入力します。認証情報管理がセルフマネージド（Self-managed）にします。

    EC2 接続をセットアップ（Setup）して、「[1.1.2 作成](# 1.1.2 作成)」でインスタンスの「jay-ec2-01」を選択します。

    他のすべてのデフォルト。長く待って…ついに作成が完了します。

    2.2.3 使用

    「パブリックアクセス可能：なし」の原因：データベース サービスは通常、セキュリティ（Security）上の理由から一般に公開されていないため、データベース サービスにアクセスするには、選択した EC2 インスタンスを使用する必要があります。

    データベースの詳細をクリックして、アクセス用のエンドポイントとポートを表示します。

    設定の中で「DB 名」がある。

    シェルでインスタンスを操作して、データベースに訪問します。
