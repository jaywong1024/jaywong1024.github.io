    Java 语言提供了八种基本类型，六种数字类型（四个整数型，两个浮点型），一种布尔型，还有一种字符类型

    类型 大小（bit） 存储数据 最小值 最大值 默认值 包装器类型

    二、修饰符

    2.1 访问控制修饰符

    Java 中，可以使用访问控制符来保护对类、变量、方法和构造方法的访问。Java 支持 4 种不同的访问权限

    • default (即默认，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法
    • private : 在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）
    • public : 对所有类可见。使用对象：类、接口、变量、方法
    • protected : 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）

    我们可以通过以下表来说明访问权限：

    修饰符 当前类 同一包内 子孙类(同一包) 子孙类(不同包) 其他包
    public Y Y Y Y Y
    protected Y Y Y N N
    default Y Y Y N N
    private Y N N N N

    2.2 非访问修饰符

    2.3.1 static 修饰符

    • 静态变量：
      • static 关键字用来声明独立于对象的静态变量，无论一个类实例化多少对象，它的静态变量只有一份拷贝。 静态变量也被称为类变量，局部变量不能被声明为 static 变量
    • 静态方法：
      • static 关键字用来声明独立于对象的静态方法，静态方法不能使用类的非静态变量，静态方法从参数列表得到数据，然后计算这些数据

    2.3.2 final 修饰符

    • final 方法：
      • 父类中的 final 方法可以被子类继承，但是不能被子类重写
    • final 类：
      • final 类不能被继承，没有类能够继承 final 类的任何特性

    2.3.3 abstract 修饰符

    • 抽象类：

      • 抽象类不能用来实例化对象，声明抽象类的唯一目的是为了将来对该类进行扩充
      • 一个类不能同时被 abstract 和 final 修饰，如果一个类包含抽象方法，那么该类一定要声明为抽象类，否则将出现编译错误
      • 抽象类可以包含抽象方法和非抽象方法

    • 抽象方法

      • 抽象方法是一种没有任何实现的方法，该方法的的具体实现由子类提供
      • 抽象方法不能被声明成 final 和 static
      • 任何继承抽象类的子类必须实现父类的所有抽象方法，除非该子类也是抽象类
      • 如果一个类包含若干个抽象方法，那么该类必须声明为抽象类。抽象类可以不包含抽象方法
      • 抽象方法的声明以分号结尾，例如：public abstract sample();

    2.3.4 transient 修饰符

    序列化的对象包含被 transient 修饰的实例变量时，Java 虚拟机跳过该特定的变量，这些特定的变量不会被序列化。

    2.3.5 volatile 修饰符

    volatile 修饰的成员变量在每次被线程访问时，都强制从共享内存中重新读取该成员变量的值。而且，当成员变量发生变化时，会强制线程将变化值回写到共享内存。这样在任何时刻，两个不同的线程总是看到某个成员变量的同一个值。

    6.2 StringBuilder

    根据对 String 的分析，可以得知 Spring 在大量字符操作的时候会频繁的实例化对象，所以在大量字符操作的时候推荐使用 StringBuilder，下面来看看原因

    6.2.1 数据结构

    通过 StringBuilder 源码可以发现：

    • StringBuilder 继承自 AbstractStringBuilder，实现了序列化、字符串结构接口
    • AbstractStringBuilder 通过 char 数组的方式来保存数据，但没有使用 final 修饰，所以 StringBuilder 实例化后是可以修改的
    • AbstractStringBuilder 实现了追加接口、字符串结构接口。说明 StringBuilder 是支持追加方法 append 的，并且属于字符串家庭中的一员

    6.2.2 append 方法

    追加字符串到 StringBuilder 中，不用实例化对象，减少了字符串拼接操作进行多余实例化的问题

    6.3 StringBuffer

    StringBuffer 也是为了解决字符串拼接操作频繁实例化对象的问题，但他和 StringBuilder 不一样，很多方法都使用了 synchronized 修饰保证线程安全，适合多线程场景使用

    6.3.1 数据结构

    通过 StringBuffer 源码可以发现：

    • StringBuilder 继承自 AbstractStringBuilder，实现了序列化、字符串结构接口
    • StringBuilder 的 toString 方法是返回最后一次缓存，这个属性被 transient 修复，不会被序列化
    • AbstractStringBuilder 通过 char 数组的方式来保存数据，但没有使用 final 修饰，所以 StringBuffer 实例化后是可以修改的
    • AbstractStringBuilder 实现了追加接口、字符串结构接口。说明 StringBuffer 是支持追加方法 append 的，并且属于字符串家庭中的一员

    6.3.2 线程安全方法

    为保证线程安全 StringBuffer 使用了 synchronized 修饰的方法
