    1.1.1.3【编程基础 - Java 语言 - 语言基础】接口

    参考摘抄：《Effective Java》《Java 核心技术 卷1》

    一、接口概念

    官方解释：Java 接口是一系列方法的声明，是一些方法特征的集合，一个接口只有方法的特征没有方法的实现，因此这些方法可以在不同的地方被不同的类实现，而这些实现可以具有不同的行为（功能）

    二、 接口特性

    1. 接口是对类的一组需求描述，这些类要遵从接口描述的统一格式进行定义。比如 Comparable 接口，描述了实现了此接口的类是可比较的，比如 Arrays 的 sort 方法就要求对象实现 Comparable 接口
    2. 接口不是类，不能使用 new 运算符实例化一个接口，但可以声明接口的变量 Comparable c;
    3. 可以使用 instanceof 检查某个对象是否实现了某个特定的接口 if(c instanceof Comparable) {...}
    4. 接口也可以和类一样被继承 public interface Cat extends Animal {...}
    5. 接口中的方法都自动地被设置为 public，接口中地域将自动设为 public static final

    三、为什么要用接口

    3.1 不支持多继承

    实际开发中，一个类是可能符合了多组需求描述的，比如 B 站上的开发 UP 主，他同时兼具开发和视频剪辑的能力，那我们可以定义 Development 和 VideoClip 作为能力的描述。由于 Java 不支持多继承的设计，一个类是无法继承多个抽象类的，比如以下代码会编译报错的，所以不能用抽象类（也不是完全不能用，把 Development 和 VideoClip 写在一起作为一个抽象类也可以继承，但这样抽象粒度就变大了）

    public class Person extends Development, VideoClip {...} // ERROR

    但是每个类可以像下面这样实现多个接口，所以一般使用接口对类进行需求描述，扩展性也比较强

    public class Person implements Development, VideoClip {...} // OK

    3.2 抽象和解耦

    对同一组需求进行抽象，不拘于实现细节，从而使得同一组需求具有通用性和可替代性。解耦只是抽象的附带效果，接口就是实现解耦的一种手段

    3.3 面向接口编程

    面向接口编程是一种编程思想，接口作为一组需求实体抽象出来的一种表现形式，用于抽离内部实现进行外部沟通，最终实现内部变动而不影响与外部的交互。这并不是比面向对象编程更先进的一种独立的编程思想，应该说面向接口编程属于面向对象思想体系的一部分，或者说它是面向对象编程体系的思想精髓之一

    3.4 与接口有关的软件开发原则

    • 依赖倒置原则：高层模块不应该依赖于低层模块，他们应该共同依赖于抽象，而这个抽象就是接口
    • 接口隔离原则：定义接口时，尽量使接口功能单一。例如 Comparable 和 Iterable 只定义 比较 和 迭代 功能
