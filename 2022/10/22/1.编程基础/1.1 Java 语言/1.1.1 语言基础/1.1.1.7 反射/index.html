    1.1.1.7【编程基础 - Java 语言 - 语言基础】反射

    一、概述

    1.1 简介

    能够分析类能力的程序成为反射（reflective）。反射机制的功能非常强大，它有如下功能：

    • 在运行时分析类的能力
    • 在运行时查看对象，例如，编写一个 toString 方法供所有类使用
    • 利用 Method 对象调用类方法

    反射的使用者主要是工具开发者，一般在业务代码中是使用不到的

    1.2 类介绍

    反射相关类都在 java.lang.reflect.* 包下

    类 含义
    java.lang.Class 类信息
    java.lang.reflect.Method 类中的方法
    java.lang.reflect.Constructor 类中的构造方法
    java.lang.reflect.Field 类中的成员变量（静态变量 + 实例变量）

    二、Class 类

    2.1 获取 Class

    获取 Class 类有以下三种方式

    2.2 Class 类方法

    方法名 备注
    public T newInstance() 实例化对象
    public String getName() 返回全限定名
    public String getSimpleName() 返回类名
    public Field[] getFields() 返回类中 public 修饰的属性
    public Field[] getDeclaredFields() 返回类中所有的属性
    public Field getDeclaredField(String name) 根据属性名 name 获取指定的属性
    public native int getModifiers() 返回此类或接口的修饰符，以整数编码。
    修饰符由 Java 虚拟机的公共、受保护、私有、最终、静态、抽象和接口常量组成。
    它们应该使用类 Modifier 的方法进行解码
    public Method[] getDeclaredMethods() 返回类中所有的实例方法
    public Method getDeclaredMethod(String name, Class<?>… parameterTypes) 根据方法名 name 和方法形参获取指定方法
    public Constructor<?>[] getDeclaredConstructors() 返回类中所有的构造方法
    public Constructor getDeclaredConstructor(Class<?>… parameterTypes) 根据方法形参获取指定的构造方法
    public native Class<? super T> getSuperclass() 返回调用类的父类
    public Class<?>[] getInterfaces() 返回调用类实现的接口集合

    3.2 Field 类方法

    方法名 备注
    public String getName() 获取属性名
    public int getModifiers() 返回此类或接口的修饰符，以整数编码。
    修饰符由 Java 虚拟机的公共、受保护、私有、最终、静态、抽象和接口常量组成。
    它们应该使用类 Modifier 的方法进行解码
    public Class<?> getType() 获取属性的 Class 类
    public void set(Object obj, Object value) 设置属性值
    public Object get(Object obj) 获取属性值
    public void setAccessible(boolean flag) 将此对象的可访问标志设置为指示的布尔值。
    值 true 表示反射对象在使用时应禁止 Java 语言访问检查。
    值 false 表示反射的对象应该强制执行 Java 语言访问检查

    3.3 私有域属性的获取和设置

    在对私有域的属性进行 set 和 get 操作时，需要调用 setAccessible 将此对象的可访问标志设置为 true
