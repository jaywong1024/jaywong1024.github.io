    二、常用的函数式接口

    函数式接口一般用于方法的参数中，配合 lambda 表达式使用，不进行显示声明对象（下面声明对象是为了演示），下面来看一下我最常用的四个函数式接口

    2.1 Predicate

    Predicate 接口定义了 boolean test(T t); 方法，它接收一个泛型 T 对象，并返回一个 boolean 值

    2.2 Consumer

    Consumer 接口定义了 void accept(T t); 方法，它接收一个泛型 T 对象，没有返回值

    2.3 Function

    Function 接口定义了 R apply(T t); 方法，它接收一个泛型 T 对象，返回一个泛型 R 对象

    2.4 Supplier

    Supplier 接口定义了 T get(); 方法，它没有参数，返回一个泛型 T 对象

    四、Stream

    4.1 简介

    Stream 是 JDK 1.8 加入的新接口，主要用于处理集合操作，它是函数式编程的核心之一，可以以一种流式编程对数据进行各种加工和运算。就像工业中的流水线，将原料放到流水线中，进行操作 1、操作 2…操作 N，最后输出结果。Stream 也是如此，可以将操作分为创建流、一系列中间操作（操作 1、操作 2…操作 N）、终止流（输出结果）。

    4.3 中间操作

    中间操作分为 无状态操作 和 有状态操作。无状态操作不需要依赖额外的空间存放临时结果；有状态操作则需要额外空间

    对象 相关方法 / 描述
    无状态操作 map / mapToXxx 对流中的元素逐一执行函数并返回另一个流
    flatMap / flatMapToXxx 对流中的数组/列表元素进行平铺合并为一个流返回
    filter 对流中的元素逐一进行过滤操作
    peek 对流中的元素逐一执行函数并返回自身
    unordered 消除对流必须保持有序的约束，从而允许后续操作使用不必考虑排序的优化
    有状态操作 distinct 返回由该流的不同元素（根据 Objecte#quals 判断）组成的流
    sorted 对流中的元素进行排序后返回一个新的流（可以自行传入迭代器）
    limit 返回由该流的元素进行截断后组成的流，截断长度不大于 maxSize
    skip 在丢弃流的前 n 个元素之后，返回由该流的其余元素组成的流

    4.4 终止流

    终止操作分为 非短路操作 和 短路操作。非短路操作可能返回一个或多个元素，但必定使用到了所有元素；短路操作则不必运用所有元素，有结果直接返回

    对象 相关方法 / 描述
    非短路操作 forEach 遍历流中的所有元素
    forEachOrdered 遍历流中的所有元素（如果有定义排序则有序遍历）
    collect 根据收集器的转换方法得到想要的数据结构（通常是 List 和 Map）
    toArray 将流中的所有元素整理成 Object[] 返回
    reduce 根据一定的规则将流中的元素进行计算后返回一个唯一的值
    min / max / count 获取流中所有元素的 最小值、最大值、总数
    短路操作 findFirst / findAny 返回流中的第一个元素 / 随机返回流中任何一个元素
    allMatch / anyMatch / noneMatch 对流中的元素执行 java.util.function.Predicate#test 方法进行匹配，当出现结果则中断执行并返回 boolean 结果。上面三个方法分别对应：所有元素的执行结果所有都为 true；任意一个元素的执行结果为 true；所有元素的执行结果都为 false
